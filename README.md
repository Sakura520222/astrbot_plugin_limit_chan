# AI 使用次数限制插件

一个强大的 AstrBot 插件，用于限制用户每日 AI 使用次数，支持多级配置、黑白名单和独立/共享模式。

## ✨ 功能特性

- ✅ **多级配置系统**：支持全局、群组、用户三级配置
- ✅ **黑白名单机制**：优先级黑名单 > 白名单
- ✅ **灵活的计数模式**：独立模式（用户独立计数）和共享模式（群组共享计数）
- ✅ **高并发支持**：配置文件读取 + SQLite WAL 模式，极致性能
- ✅ **自动迁移**：启动时自动将旧数据库配置迁移到配置文件
- ✅ **完整的管理指令**：提供丰富的配置和管理功能
- ✅ **私聊和群聊支持**：自动识别私聊和群聊场景

## 🚀 性能优化

### v2.0 重大更新

- 📁 **配置迁移至文件**：黑名单、白名单、用户/群组配置从数据库迁移到配置文件
- ⚡ **零延迟配置读取**：配置从内存读取，无数据库查询开销
- 🔥 **减少锁竞争**：数据库仅处理使用记录，提升高并发性能
- 🔄 **自动数据迁移**：首次启动自动检测并迁移旧配置，无需手动操作

| 操作 | 优化前 | 优化后 |

## 📦 安装

1. 将 `astrbot_plugin_limit_chan` 文件夹放入 AstrBot 的 `plugins` 目录
2. 重启 AstrBot 或使用插件管理器加载插件
3. 插件会自动创建数据库并初始化配置
4. 如有旧配置，会自动迁移到配置文件

## 🎯 配置优先级

插件按以下优先级检查配置（从高到低）：

1. **黑名单** → 直接拦截，不允许使用
2. **白名单** → 无限制使用
3. **用户特定配置** → 使用用户专属配置
4. **群组特定配置** → 使用群组配置（仅群聊）
5. **全局默认配置** → 使用全局默认值

## 📝 配置方式

### 通过 AstrBot 管理界面配置

推荐通过 AstrBot 的插件管理界面编辑配置：

#### 1. 黑名单配置

```json
{
  "blacklist": {
    "qq": ["用户ID1", "用户ID2"],
    "wechat": ["用户ID3"]
  }
}
```

#### 2. 白名单配置

```json
{
  "whitelist": {
    "qq": ["VIP用户1", "VIP用户2"]
  }
}
```

白名单用户可无限制使用 AI 功能。

#### 3. 用户配置

```json
{
  "user_configs": {
    "qq": {
      "用户ID1": {
        "daily_limit": 50
      },
      "用户ID2": {
        "daily_limit": 100
      }
    }
  }
}
```

#### 4. 群组配置

```json
{
  "group_configs": {
    "qq": {
      "群组ID1": {
        "daily_limit": 30,
        "mode": "shared"
      },
      "群组ID2": {
        "daily_limit": 20,
        "mode": "individual"
      }
    }
  }
}
```

#### 5. 全局配置

```json
{
  "daily_limit": 20,
  "mode": "individual"
}
```

- `daily_limit`：每日限制次数（-1 表示无限制）
- `mode`：`individual`（独立模式）或 `shared`（共享模式）

## 💬 使用指令

### 普通用户指令

#### 查询使用情况
```
/limit
```
显示今日已使用次数、剩余次数、模式和配置来源。

### 管理员指令

#### 查看黑名单
```
/limit blacklist list
```
显示当前平台的黑名单列表（从配置文件读取）。

#### 查看白名单
```
/limit whitelist list
```
显示当前平台的白名单列表（从配置文件读取）。

#### 设置全局配置
```
/limit global daily_limit 20
/limit global mode individual
```
- `daily_limit`：全局默认限制次数
- `mode`：全局默认模式（individual/shared）

#### 重置计数
```
/limit reset <用户ID|群ID>
```
重置指定用户或群组的使用计数。

#### 查看统计
```
/limit stats [用户ID|群ID]
```
查看使用统计信息。不指定参数则显示所有记录（最近50条）。

#### 清空使用记录
```
/limit clear_db
```
**⚠️ 警告：此操作将清空所有使用记录！**

执行此命令后：
1. 系统会显示警告信息
2. 需要在 30 秒内发送「确认」来确认操作
3. 发送其他内容或超时将自动取消操作

**注意：黑名单、白名单、用户/群组配置已移至配置文件，不受此操作影响。**

## 🔄 数据迁移

插件启动时会自动检测旧配置并迁移：

1. 检测数据库中的旧表（blacklist、whitelist、user_config、group_config）
2. 将数据迁移到内存中的配置对象
3. 删除旧表，释放数据库空间
4. 提示管理员通过管理界面保存配置

**迁移后的数据需要通过 AstrBot 管理界面保存才能持久化。**

## 📊 计数模式说明

### 独立模式（individual）
每个用户独立计数，互不影响。

**场景示例：**
```
群聊X（独立模式，限制20次/天）
- 用户A使用了5次
- 用户B使用了3次
- 用户C还可以使用20次（不受其他用户影响）
```

### 共享模式（shared）
群组内所有用户共用一个计数。

**场景示例：**
```
群聊Y（共享模式，限制50次/天）
- 用户A使用了10次
- 用户B使用了15次
- 全群已使用25次，剩余25次
- 用户C再使用25次后，全群都将达到上限
```

## 🗄️ 数据存储

### 配置文件
黑名单、白名单、用户/群组配置存储在插件配置文件中，通过 AstrBot 管理界面编辑。

### 数据库
使用记录存储在 SQLite 数据库中，数据文件位于：
```
plugins/astrbot_plugin_limit_chan/data/astrbot_plugin_limit_chan.db
```

### 数据库表结构

- `global_config`：全局配置（兼容保留）
- `ai_usage`：使用记录

## ⚙️ 技术特性

- **零延迟配置读取**：配置从内存读取，无数据库查询
- **高并发支持**：使用 WAL 模式和异步操作
- **原子性操作**：使用 UPSERT 语法保证计数准确性
- **索引优化**：针对查询场景优化数据库索引
- **异步处理**：计数更新不阻塞 AI 请求
- **自动迁移**：无缝升级，无需手动操作

## ⚠️ 注意事项

1. **权限要求**：所有管理指令需要管理员权限
2. **跨平台**：配置按平台隔离（QQ、微信等独立配置）
3. **自动重置**：每日使用次数会在新的一天自动重置
4. **配置持久化**：修改配置后需通过管理界面保存
5. **数据备份**：建议定期备份数据库文件

## ❓ 常见问题

### Q: 如何查看用户的用户ID？
A: 可以使用 AstrBot 的用户信息查询指令，或在日志中查看。

### Q: 配置文件和数据库有什么区别？
A: 黑名单、白名单、用户/群组配置在配置文件中（快速读取），使用记录在数据库中（持久化存储）。

### Q: 共享模式下如何区分用户？
A: 共享模式下，所有用户共用群组的计数，但系统会记录每个用户的使用情况用于统计。

### Q: 黑名单和白名单可以同时存在吗？
A: 可以，但黑名单优先级更高，在黑名单中的用户即使也在白名单中也无法使用。

### Q: 如何禁用某个群组的限制？
A: 可以将该群组的 `daily_limit` 设置为 -1（无限制），或者将用户加入白名单。

### Q: 旧版本的数据会丢失吗？
A: 不会，插件启动时会自动迁移旧配置到配置文件，并保留所有使用记录。

### Q: 数据库文件损坏怎么办？
A: 删除 `astrbot_plugin_limit_chan.db` 文件，重启插件会自动重建数据库（仅丢失使用记录，配置不受影响）。
