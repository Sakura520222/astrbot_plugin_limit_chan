# AI 使用次数限制插件

一个强大的 AstrBot 插件，用于限制用户每日 AI 使用次数，支持多级配置、黑白名单和独立/共享模式。

## 功能特性

- ✅ **多级配置系统**：支持全局、群组、用户三级配置
- ✅ **黑白名单机制**：优先级黑名单 > 白名单
- ✅ **灵活的计数模式**：独立模式（用户独立计数）和共享模式（群组共享计数）
- ✅ **高并发支持**：基于 SQLite + WAL 模式，支持高并发场景
- ✅ **完整的管理指令**：提供丰富的配置和管理功能
- ✅ **私聊和群聊支持**：自动识别私聊和群聊场景

## 安装

1. 将 `astrbot_plugin_limit_chan` 文件夹放入 AstrBot 的 `plugins` 目录
2. 重启 AstrBot 或使用插件管理器加载插件
3. 插件会自动创建数据库并初始化配置

## 配置优先级

插件按以下优先级检查配置（从高到低）：

1. **黑名单** → 直接拦截，不允许使用
2. **白名单** → 无限制使用
3. **用户特定配置** → 使用用户专属配置
4. **群组特定配置** → 使用群组配置（仅群聊）
5. **全局默认配置** → 使用全局默认值

## 使用说明

### 普通用户指令

#### 查询使用情况
```
/limit
```
显示今日已使用次数、剩余次数、模式和配置来源。

### 管理员指令

所有管理指令需要管理员权限。

#### 设置全局配置
```
/limit global daily_limit 20
/limit global mode individual
```
- `daily_limit`：全局默认限制次数
- `mode`：全局默认模式（individual/shared）

#### 设置群组配置
```
/limit group <群ID> <限制次数> [模式]
```
示例：
```
/limit group 123456 50 shared
/limit group 123456 30 individual
```

#### 设置用户配置
```
/limit user <用户ID> <限制次数>
```
示例：
```
/limit user 789012 100
```

#### 黑名单管理
```
/limit blacklist add <用户ID> [理由]
/limit blacklist remove <用户ID>
/limit blacklist list
```
示例：
```
/limit blacklist add 789012 违规使用
/limit blacklist remove 789012
/limit blacklist list
```

#### 白名单管理
```
/limit whitelist add <用户ID>
/limit whitelist remove <用户ID>
/limit whitelist list
```

#### 重置计数
```
/limit reset <用户ID|群ID>
```
重置指定用户或群组的使用计数。

#### 查看统计
```
/limit stats [用户ID|群ID]
```
查看使用统计信息。不指定参数则显示所有记录（最近50条）。

## 计数模式说明

### 独立模式（individual）
每个用户独立计数，互不影响。

**场景示例：**
```
群聊X（独立模式，限制20次/天）
- 用户A使用了5次
- 用户B使用了3次
- 用户C还可以使用20次（不受其他用户影响）
```

### 共享模式（shared）
群组内所有用户共用一个计数。

**场景示例：**
```
群聊Y（共享模式，限制50次/天）
- 用户A使用了10次
- 用户B使用了15次
- 全群已使用25次，剩余25次
- 用户C再使用25次后，全群都将达到上限
```

## 配置示例

### 场景1：私聊用户
```
用户A私聊机器人 → 使用用户A的独立配置（如果有）或全局配置
```

### 场景2：群组-独立模式
```
群聊X设置为独立模式，30次/天
每个群成员独立计数，互不影响
```

### 场景3：群组-共享模式
```
群聊Y设置为共享模式，100次/天
全群共用100次额度，用完后所有人无法使用
```

### 场景4：黑名单用户
```
用户A在黑名单 → 无论在哪里都无法使用 AI 功能
```

### 场景5：白名单用户
```
用户B在白名单 → 无限制使用 AI 功能，不受任何配置影响
```

### 场景6：用户特定配置
```
用户C配置了50次/天
无论在哪个群组或私聊，都使用50次限制
```

## 数据存储

插件使用 SQLite 数据库存储所有数据，数据文件位于：
```
plugins/astrbot_plugin_limit_chan/data/astrbot_plugin_limit_chan.db
```

### 数据库表结构

- `global_config`：全局配置
- `group_config`：群组配置
- `user_config`：用户配置
- `blacklist`：黑名单
- `whitelist`：白名单
- `ai_usage`：使用记录

## 技术特性

- **高并发支持**：使用 WAL 模式和异步操作
- **原子性操作**：使用 UPSERT 语法保证计数准确性
- **索引优化**：针对查询场景优化数据库索引
- **异步处理**：计数更新不阻塞 AI 请求

## 注意事项

1. **权限要求**：所有管理指令需要管理员权限
2. **跨平台**：配置按平台隔离（QQ、微信等独立配置）
3. **自动重置**：每日使用次数会在新的一天自动重置
4. **数据备份**：建议定期备份 `astrbot_plugin_limit_chan.db` 文件

## 常见问题

### Q: 如何查看用户的用户ID？
A: 可以使用 AstrBot 的用户信息查询指令，或在日志中查看。

### Q: 共享模式下如何区分用户？
A: 共享模式下，所有用户共用群组的计数，但系统会记录每个用户的使用情况用于统计。

### Q: 黑名单和白名单可以同时存在吗？
A: 可以，但黑名单优先级更高，在黑名单中的用户即使也在白名单中也无法使用。

### Q: 如何禁用某个群组的限制？
A: 可以将该群组设置为很大的限制次数，或者将所有用户加入白名单。

### Q: 数据库文件损坏怎么办？
A: 删除 `astrbot_plugin_limit_chan.db` 文件，重启插件会自动重建数据库。
